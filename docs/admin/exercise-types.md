# Админка типов упражнений

## Обзор
Раздел `Типы упражнений` в админ-панели позволяет управлять каталогом типов упражнений и их схемами полей. Доступ осуществляется по адресу `/admin/exercise-types` (требуется административная роль).

## Возможности списка
- Просмотр всех типов с иконкой, описанием, количеством полей и упражнений.
- Быстрый поиск по имени, ключу, домену и описанию.
- Фильтр «Показывать неактивные» для скрытых типов.
- Кнопка `Создать тип` для перехода на форму создания (`/admin/exercise-types/new`).
- Переключение активности через кнопку `Скрыть/Показать` с мгновенным состоянием и уведомлением.
- Переход к карточке типа (`/admin/exercise-types/{id}`) по кнопке `Открыть`.

## Карточка типа (`/admin/exercise-types/{id}`)
### Основные данные
- Редактирование полей: `Название`, `Ключ`, `Домен`, `Иконка`, `Порядок`, `Описание`.
- Кнопка `Сохранить изменения` отправляет PUT-запрос; при успехе появляется тост «Изменения сохранены».
- Кнопка `Скрыть/Сделать активным` переключает статус `is_active` с уведомлением.
- Кнопка `Удалить` удаляет тип (доступно только если нет связанных упражнений), подтверждение через `confirm()`.

### Поля типа
- Форма создания поля включает:
  - `Заголовок поля` (обязательное).
  - `Ключ` (обязательное, пробелы автоматически заменяются на `_`).
  - `Тип` (string, text, integer, number, boolean, enum, array_enum, json).
  - Дополнительные атрибуты: `Default (JSON)`, `Опции (JSON)`, `Минимум`, `Максимум`, `Шаг`, `Подсказка`, флаг «Поле обязательное».
- Поля `Default` и `Опции` принимают корректный JSON. При ошибке отображается предупреждение.
- В таблице ниже отображаются существующие поля с порядком, типом, обязательностью и значением по умолчанию.
- Каждое поле можно удалить через кнопку `Удалить` (с подтверждением). Успешная операция сопровождается тостом «Поле удалено».

### Использование
- Вкладка «Использование» показывает количество упражнений, связанных с типом, и время последнего обновления.

## API-хелперы
Фронтенд использует функции из `frontend/src/lib/api.ts`:
- `fetchAdminExerciseTypes` — список типов.
- `fetchAdminExerciseTypeDetail` — детальная карточка.
- `createExerciseType`, `updateExerciseType`, `deleteExerciseType` — управление типами.
- `createExerciseTypeField`, `deleteExerciseTypeField` — управление дополнительными полями.

## Тестирование
Автоматические сценарии Playwright находятся в `frontend/tests/e2e/admin-exercise-types.spec.ts`:
- Переключение активности на списке.
- Редактирование основного описания.
- Создание и удаление пользовательского поля.
Запуск: `cd frontend && npx playwright test admin-exercise-types.spec.ts`.

## Требования к данным
Для локальной разработки используйте сидер `ExerciseTypesSeeder` (`app/database/seeders/ExerciseTypesSeeder.php`), который наполняет таблицы типов и их полей.
