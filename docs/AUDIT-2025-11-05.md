<details>
<summary>Шаблон CI конфига</summary>

```yaml
name: CI

on: [push, pull_request]

jobs:
  backend:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_PASSWORD: secret
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
    steps:
      - uses: actions/checkout@v4
      - uses: shivammathur/setup-php@v2
        with:
          php-version: '8.3'
          extensions: pgsql, pdo_pgsql
      - run: cd app && composer install
      - run: cd app && ./vendor/bin/pint --test
      - run: cd app && php artisan test

  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: cd frontend && npm ci
      - run: cd frontend && npm run lint
      - run: cd frontend && npm test
```
</details>

**DoD:**
- CI запускается на каждый push
- Все чеки проходят
- Badge в README (когда появится)

## 🎯 Рекомендации по процессу

### 1. Установить WIP Limit

**Правило:** Максимум 2-3 задачи in_progress одновременно

**Почему:**
- 7 задач in_progress = 0 завершенных задач
- Context switching снижает продуктивность
- Лучше 1 завершенная, чем 7 в процессе

### 2. Daily/Weekly sync

**Формат:**
- Async update в трекере (комментарии к задачам)
- Или 15-min daily standup
- Обязательно фиксировать блокеры

### 3. Definition of Done (DoD)

**Для каждой задачи обязательно:**
- [ ] Код написан
- [ ] Тесты добавлены (минимум 1)
- [ ] CI проходит
- [ ] Документация обновлена (если нужно)
- [ ] Code review пройден
- [ ] Задача обновлена в трекере

### 4. Tech Debt Sprint

**Предложение:** Каждый 4-й спринт = техдолг

**Цель:**
- Не накапливать долг
- Регулярно улучшать инфраструктуру
- Обновлять зависимости
