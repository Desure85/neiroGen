# Задание: унификация UI и вынос общих компонентов

## Цель
Обеспечить единый визуальный стиль кабинетов администратора и терапевта, сократить дублирование верстки, вынести повторяющиеся фрагменты в переиспользуемые компоненты.

## Компоненты к реализации
- **DashboardHeader** — шапка разделов с бейджем, заголовком, подзаголовком и правым слотом под действия.
- **DashboardTabsGroup** — обертка над `Tabs` с `DashboardTabsList`, управление значением и стилями (поддержка `primary`/`secondary`).
- **DashboardStatCard** — карточка метрики с иконкой, подписью и значением.
- **SectionCard** — контейнер на базе `Card` с заголовком, описанием и секцией действий.
- **ListHeader** — шапка таблиц/списков с заголовком, фильтрами и CTA (например, "Создать тип").

## Задачи
1. **Инвентаризация**
   - Найти повторяющиеся блоки в `frontend/src/app/admin/` и `frontend/src/app/therapist/`.
   - Зафиксировать, какие пропсы нужны для универсальности каждого компонента.
2. **Создание компонентов**
   - Разместить новые компоненты в `frontend/src/components/`.
   - Обеспечить поддержку темной темы и адаптивности (используем tailwind).
3. **Рефакторинг страниц**
   - Обновить `admin/layout.tsx` и `therapist/page.tsx`, заменив дублирующиеся `<div>` на новые компоненты.
   - Применить `DashboardHeader` и `DashboardTabsGroup` во всех верхних секциях.
   - Использовать `SectionCard` и `ListHeader` в `admin/page.tsx`, `admin/exercise-types`, `therapist/page.tsx`.
4. **Проверки**
   - Прогнать `npm run lint` и `npx playwright test tests/e2e/admin-exercise-types.spec.ts`.
   - Сделать визуальную проверку `http://localhost:3001/admin` и `.../therapist`.
5. **Документация**
   - Обновить README или соответствующий раздел в `/docs/`, указав новые компоненты и паттерны использования.

## Критерии готовности
- Все таверны/карточки используют новые компоненты без дублирования JSX.
- Визуальный стиль кабинетов совпадает (цвета табов, отступы, фоновые карточки).
- Playwright и линтер проходят без ошибок.
- Документация отражает новые компоненты и их API.
