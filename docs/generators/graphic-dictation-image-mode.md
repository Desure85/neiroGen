# Генерация графического диктанта из изображения

## Обзор

Режим **"Из изображения"** использует контурную трассировку для преобразования любого изображения в графический диктант. Это лучше чем LLM для точных фигур!

## Почему "Из изображения" лучше чем LLM?

| Параметр | LLM | Изображение |
|----------|-----|-------------|
| Точность | ❌ Низкая | ✅ Высокая |
| Узнаваемость | ❌ Абстрактная | ✅ Реалистичная |
| Контроль | ❌ Случайный | ✅ Полный |
| Скорость | 🐌 5-15 сек | ⚡ 1-3 сек |

**Вывод:** Для технических объектов (кран, экскаватор) используйте изображения!

## Как это работает

1. **Загрузка** - вы даёте URL или base64 изображения
2. **Порог** - изображение преобразуется в чёрно-белое (бинаризация)
3. **Контур** - алгоритм находит границы фигуры
4. **Упрощение** - Douglas-Peucker удаляет лишние точки
5. **Сглаживание** - опционально сглаживает острые углы
6. **Масштабирование** - подгоняет под размер сетки

## Параметры настройки

### Simplification (упрощение)
**Диапазон:** 0.5 - 3.0  
**По умолчанию:** 1.5

- **0.5** - максимум деталей, много шагов (50-100+)
- **1.5** - баланс деталей и простоты (20-50 шагов)
- **3.0** - очень упрощённо, мало шагов (10-20)

**Рекомендация:**
- Для детей 6-8 лет: `2.0-3.0`
- Для детей 9-12 лет: `1.0-2.0`
- Для подростков: `0.5-1.0`

### Smoothing (сглаживание)
**Диапазон:** 0 - 5  
**По умолчанию:** 0 (выключено)

- **0** - без сглаживания, сохраняет острые углы
- **1-2** - лёгкое сглаживание, убирает мелкие зигзаги
- **3-5** - сильное сглаживание, более плавные контуры

**Рекомендация:**
- Для пиксельной графики: `0` (без сглаживания)
- Для фотографий: `2-3` (убирает артефакты)
- Для рисунков: `1` (лёгкое сглаживание)

## Рекомендации по изображениям

### ✅ Хорошие изображения:
- **Силуэты** - чёрная фигура на белом фоне
- **Иконки** - простые векторные картинки
- **Контурные рисунки** - чёткие границы
- **Пиксель-арт** - уже в формате сетки
- **Размер:** 200x200 - 1024x1024 пикселей
- **Формат:** PNG, JPEG, WebP

### ❌ Плохие изображения:
- Фотографии (много деталей)
- Градиенты (нет чётких границ)
- Текст (слишком сложно)
- Размытые изображения
- Слишком маленькие (<100px)
- Слишком большие (>2048px)

## Алгоритм работы (технические детали)

### 1. Preprocessing
```
Image → Grayscale → Threshold (127) → Binary Mask
```

### 2. Oversampling (4x)
```
Grid 20×20 → Oversample to 80×80 → Better precision
```

### 3. Contour Extraction
```
Moore-Neighbor Tracing → Extract boundary pixels
```

### 4. Douglas-Peucker Simplification
```
100 points → (epsilon=1.5*4) → 25 points
```

### 5. Smoothing (optional)
```
Weighted average: (prev + 2*current + next) / 4
Iterations: 0-5
```

### 6. Downscale
```
80×80 → Scale /4 → 20×20 grid
```

### 7. Command Generation
```
Points → Directions (up/down/left/right/diagonals)
```

## Troubleshooting

### Слишком много шагов (>80)

**Причина:** Слишком детальное изображение или низкое упрощение

**Решение:**
- Увеличьте `simplification` до 2.0-3.0
- Уменьшите размер сетки
- Используйте более простое изображение

### Фигура неузнаваема

**Причина:** Слишком сильное упрощение

**Решение:**
- Уменьшите `simplification` до 0.8-1.2
- Увеличьте размер сетки (25×25 или 30×30)
- Добавьте `smoothing: 1-2`

### Острые углы и зигзаги

**Причина:** Пиксели на контуре

**Решение:**
- Добавьте `smoothing: 2-3`
- Используйте векторное изображение вместо растрового

### Пустой результат

**Причина:** Изображение полностью белое или чёрное

**Решение:**
- Проверьте контрастность изображения
- Убедитесь что есть чёткая фигура на фоне
- Попробуйте инвертировать цвета
