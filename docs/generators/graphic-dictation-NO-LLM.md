# Почему LLM удалён из графического диктанта

## TL;DR

**LLM плохо справляется с пространственными задачами и генерацией координат.**

Вместо LLM используйте:
1. **14 готовых шаблонов** - идеальное качество
2. **Режим "Из изображения"** - точная трассировка контура

## Проблемы LLM для графических диктантов

### 1. Низкая точность
LLM не "видит" сетку и не может представить как фигура будет выглядеть. Результат - случайные координаты.

**Пример:**
- Запрос: "нарисуй подъёмный кран"
- Результат: абстрактная фигура, не похожая на кран

### 2. Непредсказуемость
Один и тот же запрос даёт разные результаты:
- Попытка 1: что-то похожее на собаку
- Попытка 2: непонятная абстракция
- Попытка 3: случайные зигзаги

### 3. Медленная генерация
- LLM: 5-15 секунд
- Шаблоны: мгновенно
- Изображения: 1-3 секунды

### 4. Стоимость
- Groq (бесплатно) - лимиты 6000 токенов/мин
- OpenAI - ~$0.0001 за фигуру
- Шаблоны/Изображения - бесплатно без лимитов

## Что мы пробовали

### Попытка 1: Простой промпт
```
"Создай фигуру на сетке"
```
**Результат:** Полностью случайные координаты

### Попытка 2: Детальный промпт
```
"Ты эксперт по графическим диктантам.
Фигура должна быть узнаваемой.
Примеры: кран = стрела + кабина + противовес"
```
**Результат:** Чуть лучше, но всё ещё неузнаваемо

### Попытка 3: Низкая temperature
```
Temperature: 0.7 → 0.3
```
**Результат:** Более предсказуемо, но качество не улучшилось

### Вывод
LLM фундаментально не подходит для этой задачи. Нужны другие подходы.

## Правильные подходы

### ✅ 1. Готовые шаблоны (14 фигур)

**Преимущества:**
- Идеально нарисованы программистом
- Гарантированно узнаваемы
- Мгновенная генерация
- Адаптируются под размер сетки

**Список:**
- Простые: квадрат, прямоугольник, домик, дерево
- Средние: машинка, лодка, робот, кран, самолёт, ракета, бабочка
- Сложные: собачка, кошка, бетономешалка

**Использование:**
```json
{
  "description": "кран",
  "grid_width": 20,
  "grid_height": 20
}
```

### ✅ 2. Из изображения

**Преимущества:**
- Любая фигура
- Полный контроль через параметры
- Точная трассировка контура
- Алгоритм Douglas-Peucker для упрощения

**Параметры:**
- `simplification`: 0.5-3.0 (контроль деталей)
- `smoothing`: 0-5 (сглаживание)

**Использование:**
```json
{
  "source_image": "https://example.com/icon.png",
  "grid_width": 20,
  "grid_height": 20,
  "simplification": 1.5,
  "smoothing": 1
}
```

| **Узнаваемость** | ❌ 3/10 | ✅ 10/10 | ✅ 8/10 |
| **Скорость** | ❌ 5-15 сек | ✅ <0.1 сек | ✅ 1-3 сек |
| **Контроль** | ❌ Нет | ✅ Да | ✅ Полный |
| **Предсказуемость** | ❌ Низкая | ✅ 100% | ✅ Высокая |
| **Стоимость** | ❌ Токены | ✅ Бесплатно | ✅ Бесплатно |
| **Количество фигур** | ∞ | 14 | ∞ |

## Когда что использовать

### Используйте шаблоны если:
- ✅ Фигура есть в списке 14 шаблонов
- ✅ Нужно идеальное качество
- ✅ Нужна скорость
- ✅ Для детей (гарантированно узнаваемо)

### Используйте изображения если:
- ✅ Нужна своя уникальная фигура
- ✅ Есть готовый силуэт или иконка
- ✅ Нужен полный контроль
- ✅ Хотите экспериментировать с параметрами

### НЕ используйте LLM потому что:
- ❌ Низкая точность координат
- ❌ Непредсказуемые результаты
- ❌ Не узнаваемые фигуры
- ❌ Медленно
- ❌ Шаблоны + Изображения покрывают 100% задач

## Техническое объяснение

### Почему LLM не работает

LLM обучена на **тексте**, не на **пространственных данных**:

1. **Нет визуального понимания**
   - LLM не "видит" сетку
   - Не может представить как выглядит фигура
   - Генерирует координаты "на ощупь"

2. **Нет геометрической логики**
   - Не понимает "контур должен быть замкнут"
   - Не понимает "соседние точки должны быть рядом"
   - Может сгенерировать точки в разных частях сетки

3. **Проблема embedding space**
   - В обучающих данных мало примеров "координата X,Y → фигура"
   - LLM не обучена на графических диктантах
   - Нет переноса знаний из текстовых описаний в координаты

### Что работает

**Алгоритмический подход:**
- Шаблоны: код напрямую генерирует правильные координаты
- Изображения: computer vision алгоритмы (contour tracing, Douglas-Peucker)

**Эти методы:**
- ✅ Детерминированы
- ✅ Предсказуемы
- ✅ Быстрые
- ✅ Точные

## Заключение

**LLM - неправильный инструмент для задачи генерации координат.**

Это как использовать молоток для вкручивания шурупов - технически возможно, но есть лучшие инструменты.

### Правильное решение:
1. **Шаблоны** для стандартных фигур
2. **Изображения** для кастомных фигур
3. **NO LLM** - экономия времени и нервов

### Результат:
- ✅ 100% узнаваемых фигур
- ✅ Мгновенная генерация
- ✅ Полный контроль
- ✅ Без случайностей
- ✅ Бесплатно и без лимитов

**Шаблоны + Изображения = идеальное решение для графических диктантов!** 🎯
