### 2. Исправление дубликатов при замыкании контура

**Файл:** `frontend/src/components/graphic-dictation-editor.tsx`

#### Проблема:
При клике на первую точку/стартовую точку для замыкания контура можно было бесконечно добавлять дубликаты.

#### Решение:
- Добавлена проверка `isAlreadyClosed` перед замыканием контура
- Если последняя точка уже совпадает с первой — контур считается закрытым
- При повторном клике на замкнутый контур ничего не происходит
- Добавлена **фиолетовая подсказка** 🔒 "Контур замкнут!"

### 3. Перемещение начальной точки

**Файл:** `frontend/src/components/graphic-dictation-editor.tsx`

#### Функционал:
- Начальная точка теперь отображается как **красная точка с буквой "S"**
- Первая точка пути теперь **зелёная** (отличается от начальной)
- **Умная логика клика на "S":**
  - Если путь начинается с "S" (≥2 точки) → замыкает контур
  - Если путь НЕ начинается с "S" → включает режим перемещения
- **В режиме перемещения:** клик на любой узел сетки переносит начальную точку
- **Alt + клик** на узел → мгновенное перемещение начальной точки (всегда)

#### Визуальные индикаторы:
- Обычное состояние: красная точка (#ef4444)
- Режим перемещения: оранжевая точка (#f59e0b) + оранжевый баннер
- Путь начинается с "S": зелёный баннер-подсказка "можно замкнуть"
- Курсор меняется на `cursor-move` в режиме перемещения

#### Обновлённые подсказки:
```
💡 Подсказка:
- Красная точка "S" — начальная позиция диктанта
- Кликните на любой узел сетки, чтобы начать рисование пути
- Первая точка пути будет зелёной
- Кликните на существующую точку пути, чтобы удалить её
- Чтобы замкнуть контур: кликните на первую (зелёную) точку или на "S" если путь начинается с неё
- Чтобы переместить "S": кликните на "S" (если путь не начинается с неё) или Alt+клик
```

- 🔸 Оранжевый баннер: "Режим перемещения начальной точки"
- ✓ Зелёный баннер: "Путь начинается с точки S — кликните на S чтобы замкнуть"

### 4. Фиксация масштаба и положения фонового изображения-подложки

**Проблема:** При изменении размера сетки или масштабирования изображение смещалось и терялась привязка к узлам.

**Решение:**
- Добавлен `useEffect` на отслеживание `gridWidth`, `gridHeight`, `cellSize`, `imageScale`
- Изображение автоматически **центрируется** при изменении любого из этих параметров
- Функция `centerImage()` пересчитывает позицию относительно новых размеров сетки

**Код:**
```typescript
useEffect(() => {
  if (backgroundImage) {
    centerImage()
  }
}, [gridWidth, gridHeight, cellSize, imageScale])
```

### 5. Превью линии и автоматическая "лесенка" для диагоналей ✨

**Файл:** `frontend/src/components/graphic-dictation-editor.tsx`

#### Проблема:
- Не было видно как пройдёт линия до клика
- Диагонали не проходящие через узлы (например, 3 вправо + 2 вверх) рисовались напрямую

#### Решение:

**1. Превью линии при наведении**
- При наведении на узел показывается **пунктирная линия** от последней точки
- 🟢 **Зелёная** — прямая линия или чистая диагональ (dx === dy)
- 🟠 **Оранжевая** — "лесенка" (диагональ не через узлы)

**2. Умная обработка диагоналей**
```typescript
function getPathType(from: Point, to: Point) {
  const dx = Math.abs(to.col - from.col)
  const dy = Math.abs(to.row - from.row)
  
  if (dx === 0) return 'vertical'
  if (dy === 0) return 'horizontal'
  if (dx === dy) return 'diagonal'      // ✅ Рисуем напрямую
  return 'staircase'                     // ⚠️ Делаем лесенку
}
```

**3. Генерация промежуточных точек**
Если путь не идёт точно по диагонали, автоматически разбивается на 2 прямых отрезка:
```
Клик на (2, 1) от (0, 0):
↓ (0,1)     — сначала весь вертикальный отрезок
→ → (2,1)   — потом весь горизонтальный отрезок

Результат: 2 команды вместо 3
✅ "1 вниз" + "2 вправо"
❌ НЕ "1 вниз" + "1 вправо" + "1 вправо"
```

**4. Визуализация**
- Превью "лесенки" показывает все промежуточные точки полупрозрачными оранжевыми кружками
- При клике все точки автоматически добавляются в путь

**Преимущества:**
- ✅ Логопед **видит** как пройдёт линия перед кликом
- ✅ Диагонали только **идеальные** (проходят через узлы)
- ✅ "Кривые" диагонали автоматически разбиваются на команды "вправо/вверх"
- ✅ Не нужно вручную проставлять промежуточные точки

### 6. Убран двойной скролл ("окно в окне")

**Проблема:** Canvas был обёрнут в `overflow-auto`, создавая лишний скролл поверх основного.

**Решение:**
```tsx
<div className="border rounded-md p-4 bg-muted/10">
  <div className="overflow-auto max-h-[600px]">
    <canvas
      ref={canvasRef}
      className={isDraggingStart ? "cursor-move mx-auto" : "cursor-crosshair mx-auto"}
      style={{ display: "block" }}
    />
  </div>
</div>
```

- Убран `overflow-auto` с внешнего контейнера
- Добавлен внутренний контейнер с `max-h-[600px]` для ограничения высоты
- Canvas теперь `display: block` для корректного центрирования

## Как использовать

### Создание графического диктанта:

1. **Шаг 1:** Нажмите "Перейти к рисованию"
2. **Шаг 2:** 
   - Настройте размер сетки (width/height)
   - Загрузите изображение-подложку (опционально)
   - Переместите начальную точку "S" на нужную позицию (клик или Alt+клик)
   - Кликайте по узлам сетки, чтобы нарисовать путь
   - Нажмите "Сгенерировать команды"
   - Нажмите "Перейти к макету"
3. **Шаг 3:**
   - Выберите расположение инструкций (сверху/снизу/слева/справа)
   - Просмотрите предпросмотр макета
   - Нажмите "Завершить и сохранить"

### Перемещение начальной точки:

**Способ 1:** Клик на красную точку "S" → клик на новую позицию
**Способ 2:** Alt + клик на нужный узел сетки
