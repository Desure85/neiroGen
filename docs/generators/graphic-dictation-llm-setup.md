# Настройка LLM для генерации произвольных фигур

## Обзор

LLM-интеграция позволяет генерировать **любые фигуры** по текстовому описанию, не ограничиваясь библиотекой шаблонов.

**Примеры:**
- "нарисуй кран" → LLM создаёт подъёмный кран
- "создай замок" → получаем средневековый замок
- "тигр" → силуэт тигра
- "ракетную установку" → установка с ракетами

## Как это работает

1. **Пользователь** вводит описание: `"бетономешалка"`
2. **Matcher** ищет в библиотеке → **найдено** → используется шаблон ✅
3. **Пользователь** вводит: `"подъёмный кран"`
4. **Matcher** ищет в библиотеке → **не найдено**
5. **LLM Fallback** → отправляет запрос к OpenAI/Claude/Ollama
6. **LLM** генерирует координаты точек для фигуры крана
7. **Генератор** преобразует координаты в команды графического диктанта

## Настройка

### Вариант 1: OpenAI (рекомендуется)

1. Получите API ключ: https://platform.openai.com/api-keys
2. Добавьте в `.env`:
```bash
OPENAI_API_KEY=sk-proj-...
LLM_MODEL=gpt-4o-mini
```
3. Перезапустите воркер:
```bash
docker-compose restart go-graphic-dictation-worker
```

**Стоимость:** ~$0.0001 за фигуру (очень дёшево)

### Вариант 2: Ollama (бесплатно, локально)

1. Установите Ollama: https://ollama.ai/
2. Запустите модель:
```bash
ollama run llama3.1:8b
```
3. Добавьте в `.env`:
```bash
LLM_API_URL=http://host.docker.internal:11434/v1/chat/completions
LLM_MODEL=llama3.1:8b
OPENAI_API_KEY=dummy
```
4. Перезапустите воркер

**Преимущества:** Бесплатно, работает оффлайн  
**Недостатки:** Медленнее, качество хуже чем GPT-4

### Вариант 3: Claude (Anthropic)

1. Получите ключ: https://console.anthropic.com/
2. Добавьте в `.env`:
```bash
OPENAI_API_KEY=sk-ant-...
LLM_API_URL=https://api.anthropic.com/v1/messages
LLM_MODEL=claude-3-haiku-20240307
```

### Вариант 4: Без LLM (только шаблоны)

Просто не указывайте `OPENAI_API_KEY` — будут работать только готовые 13 фигур из библиотеки.

## Тестирование

### С LLM-интеграцией:

1. Откройте генератор
2. Режим: **"По описанию"**
3. Введите: `"подъёмный кран"`
4. Сложность: `medium`
5. Генерировать

**Ожидаемый результат:**
- Запрос уходит в LLM (5-10 секунд)
- Возвращается фигура крана на сетке
- Инструкции на русском: "Переместитесь в клетку...", "Двигайтесь вправо на 3..."

### Без LLM (fallback):

1. Не указывайте API ключ
2. Введите: `"подъёмный кран"` → **не найдено в шаблонах**
3. **Fallback:** система вернёт случайную фигуру из библиотеки

## Логи и отладка

### Проверка статуса LLM:

```bash
docker logs neirogen_go_gd_worker --tail 50
```

**Если LLM работает:**
```
[LLM] Generating shape "кран" via OpenAI...
[LLM] Successfully generated 32 points
```

**Если LLM не настроен:**
```
[LLM] Failed to generate shape: LLM API key not configured
[Template] Fallback to random shape: car
```

### Проверка переменных окружения:

```bash
docker exec neirogen_go_gd_worker env | grep -E "OPENAI|LLM"
```

Должно быть:
```
OPENAI_API_KEY=sk-...
LLM_MODEL=gpt-4o-mini
LLM_API_URL=https://api.openai.com/v1/chat/completions
```

## Качество результатов

### Отлично работает (GPT-4):
- Простые объекты: кран, трактор, вертолёт
- Здания: замок, башня, мост
- Животные: слон, жираф, пингвин
- Техника: экскаватор, бульдозер, погрузчик

### Средне работает (GPT-3.5, Claude):
- Сложные животные могут быть упрощены
- Абстрактные концепции требуют уточнений

### Плохо работает:
- Слишком детализированные запросы: "нарисуй Эйфелеву башню с точной архитектурой"
- Портреты людей (не для детей)

## Промпт-инжиниринг

Можете улучшить качество, отредактировав промпт в:
`/app/server/generators/graphicdictation/internal/llm/client.go`

**Текущий промпт:**
```
Ты помощник для создания графических диктантов для детей.
Создай {сложность} фигуру "{описание}" на сетке {ширина}×{высота}...
```

**Улучшенный промпт (пример):**
```
Ты эксперт по созданию обучающих материалов для детей 6-10 лет.
Создай максимально узнаваемую фигуру "{описание}"...
Фигура должна быть простой, но реалистичной.
```

## Ограничения

1. **Timeout:** 30 секунд на генерацию
2. **Max points:** До 50 точек (для удобства детей)
3. **Grid bounds:** Координаты всегда внутри сетки
4. **Rate limit:** Зависит от вашего LLM-провайдера

## Безопасность

- API ключ **не отправляется** на фронтенд
- Все запросы через Go-воркер на бэкенде
- Валидация координат перед отправкой пользователю
- Нет выполнения кода из LLM ответов
